# 万能视频下载器 - 需求分析文档

## 项目概述

### 项目名称
万能视频下载器（Universal Video Downloader）

### 项目背景
许多用户有下载保存视频到本地的需求，但部分平台不支持直接下载，或下载功能有限制（如不支持批量下载、清晰度限制等）。本项目旨在开发一个万能视频下载网站，让用户能够快速、方便地从各个平台下载视频。

### 目标用户
- 需要下载视频学习的用户
- 内容创作者（收集素材）
- 视频收藏者
- 移动端用户

## 功能需求

### 核心功能（MVP）
1. **视频信息解析**
   - 支持粘贴视频 URL
   - 自动识别视频平台
   - 展示视频标题、时长、缩略图等信息
   - 显示可选的视频格式和质量

2. **视频下载**
   - 单个视频下载
   - 支持选择视频格式（MP4、MP3、WebM 等）
   - 支持选择视频质量（1080p、720p、480p 等）
   - 实时显示下载进度

3. **多平台支持**
   - YouTube（必需）
   - Bilibili（必需）
   - TikTok、Instagram、Twitter 等（扩展）

4. **AI 学习助手（当前版本已实现）**
   - 视频总结摘要（总览、核心要点、章节摘要）
   - 字幕/转录文本展示（带时间戳）
   - 字幕文件下载（SRT/VTT/TXT）
   - 思维导图可视化
   - 思维导图全屏展示 + 高清导出（PNG/SVG）
   - AI 问答（支持流式回答）
   - AI 回答 Markdown 渲染（列表、代码块、引用、链接）

### 扩展功能（后续版本）
1. **批量下载**
   - 支持播放列表下载
   - 支持多个 URL 批量下载
   - 下载队列管理

2. **字幕功能**
   - 字幕下载
   - 字幕翻译

3. **AI 增强能力（后续）**
   - 问答多轮上下文记忆
   - 问答停止生成与重试
   - 摘要/脑图导出（Markdown/PDF）

4. **付费功能**
   - 用户系统
   - 免费版限制（每日下载次数、质量限制）
   - 付费版特权（无限下载、4K/8K、批量下载、AI 功能）

## 非功能需求

### 性能要求
- 下载速度：最大化用户带宽利用率
- 响应时间：视频信息解析 < 3 秒
- 并发支持：支持多个用户同时下载

### 兼容性要求
- 浏览器：Chrome、Firefox、Safari、Edge
- 移动端：响应式设计，支持移动浏览器
- 平台：Windows、macOS、Linux

### 安全性要求
- 请求限流：防止滥用
- 版权声明：明确仅供学习使用
- 不存储文件：下载后立即删除服务器副本

### 可用性要求
- 界面简洁：参考 ai.codefather.cn/painting 的浅色简洁风格
- 操作简单：三步完成下载（粘贴链接 -> 选择格式 -> 下载）
- 错误提示：友好的错误提示信息

## 技术约束

### 技术栈
- **前端**：Vue 3 + Vite + Tailwind CSS
- **后端**：FastAPI + yt-dlp（Python）
- **实时通信**：轮询（分析进度） + SSE（AI 问答流式响应）

### 设计约束
- UI 风格：参考 https://ai.codefather.cn/painting（浅色简洁风）
- 配色方案：
  - 主色：#0F172A（深蓝黑色）
  - 强调色：#1777FF（亮蓝色）
  - 背景：#FFFFFF（纯白）
- 轻量级：不使用数据库，临时存储

### 外部依赖
- **yt-dlp**：核心下载引擎（GitHub 14w+ Star，支持 1800+ 网站）
- **FFmpeg**：视频处理（yt-dlp 自动处理）

## 业务规则

### 下载限制（MVP 后续添加）
- 免费用户：每日 5 次下载，最高 720p
- 付费用户：无限下载，最高 4K/8K

### 平台策略
- 优先支持：YouTube、Bilibili
- 次要支持：TikTok、Instagram、Twitter
- 实验性支持：其他平台（yt-dlp 支持的所有平台）

### 风险控制
- 请求限流：防止被封号
- 不存储文件：降低法律风险
- 用户协议：版权免责声明
- 低调上线：不进行大规模宣传

## 数据模型

### VideoInfo（视频信息）
```typescript
{
  title: string           // 视频标题
  duration: number        // 时长（秒）
  thumbnail: string       // 缩略图 URL
  platform: string        // 平台名称
  uploader: string        // 上传者
  view_count: number      // 观看次数
  formats: Array<{        // 可用格式
    format_id: string
    ext: string          // 文件扩展名
    quality: string      // 质量标识（如 "1080p"）
    filesize: number     // 文件大小
  }>
}
```

### DownloadTask（下载任务）
```typescript
{
  task_id: string         // 任务 ID
  url: string            // 视频 URL
  status: string         // 状态：pending/downloading/completed/failed
  progress: number       // 进度百分比
  speed: string          // 下载速度
  eta: number           // 预计剩余时间（秒）
  file_path: string     // 下载文件路径
  error: string         // 错误信息
}
```

## API 接口设计

### 1. 获取视频信息
```
POST /api/info
Request: { "url": "https://..." }
Response: VideoInfo
```

### 2. 开始下载
```
POST /api/download
Request: { "url": "...", "format": "mp4", "quality": "1080p" }
Response: { "task_id": "uuid", "status": "processing" }
```

### 3. 获取下载状态
```
GET /api/download/status/{task_id}
Response: DownloadTask
```

### 4. 下载文件
```
GET /api/download/file/{task_id}
Response: File Download
```

### 5. 获取直链
```
POST /api/direct-url
Request: { "url": "..." }
Response: { "url": "direct_url", "success": true }
```

### 6. 启动 AI 分析任务（异步）
```
POST /api/ai/analyze/start
Request: { "url": "..." }
Response: { "task_id": "uuid", "status": "processing" }
```

### 7. 查询 AI 分析状态
```
GET /api/ai/analyze/status/{task_id}
Response: { task_id, status, stage, progress, result? }
```

### 8. 同步 AI 分析（兼容）
```
POST /api/ai/analyze
Request: { "url": "..." }
Response: VideoAnalysisResponse
```

### 9. AI 问答（流式，推荐）
```
POST /api/ai/chat/stream
Request: { "analysis_id": "uuid", "question": "..." }
Response: SSE events(start/delta/done/error)
```

### 10. AI 问答（非流式兼容）
```
POST /api/ai/chat
Request: { "analysis_id": "uuid", "question": "..." }
Response: { "answer": "...", "citations": [...] }
```

### 11. 下载字幕文件
```
GET /api/ai/transcript/download/{analysis_id}?format=srt|vtt|txt
Response: File Download
```

## 风险与限制

### 技术风险
- **平台反爬**：可能被平台封禁 IP
  - 缓解措施：请求限流、User Agent 轮换、代理池
- **yt-dlp 更新**：平台策略变化导致下载失败
  - 缓解措施：定期更新 yt-dlp 版本

### 法律风险
- **版权问题**：用户可能下载受版权保护的内容
  - 缓解措施：用户协议、免责声明、不存储文件
- **平台条款**：可能违反平台服务条款
  - 缓解措施：仅供学习使用、低调上线

### 运营风险
- **服务器成本**：大文件下载消耗带宽
  - 缓解措施：不存储文件、使用 CDN、付费功能覆盖成本
- **恶意使用**：可能被滥用
  - 缓解措施：请求限流、异常检测
